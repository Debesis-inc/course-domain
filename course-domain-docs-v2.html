§ 03 — Components
Code Reference
🎮
CourseController
@RestController · Web Layer
Handles HTTP traffic. Maps POST /api/v1/courses. Applies @Valid for Bean Validation before the method body executes. Returns ResponseEntity<CourseResponseDTO> with 201 status.
@PostMapping public ResponseEntity<CourseResponseDTO> createCourse( @Valid @RequestBody CourseRequestDTO dto) { return ResponseEntity .status(HttpStatus.CREATED) .body(courseService.createCourse(dto)); }
⚙️
CourseServiceImpl
@Service · Business Layer
Business logic layer. Maps CourseRequestDTO → Course MongoDB document → calls repository → maps result back to CourseResponseDTO. id is now String (was Long).
Course course = new Course(); course.setTitle(dto.getTitle()); course.setDescription(dto.getDescription()); Course saved = courseRepository.save(course); return new CourseResponseDTO( saved.getId(), // String ObjectId saved.getTitle(), saved.getDescription(), saved.getCreatedAt() );
🗃️
CourseRepository
MongoRepository · Data Layer
Migrated from JpaRepository<Course, Long> to MongoRepository<Course, String>. Spring Data MongoDB generates all driver calls at runtime. Zero boilerplate.
public interface CourseRepository extends MongoRepository<Course, String> { // save() → insertOne / replaceOne // findById() → find({_id:...}) // findAll() → find({}) // delete() → deleteOne }
📦
Course
@Document · MongoDB Model
Migrated from JPA @Entity to MongoDB @Document. All JPA annotations removed. @Id now from Spring Data. id is String (ObjectId). @CreatedDate auto-stamps.
@Document(collection = "course") @Data @NoArgsConstructor @AllArgsConstructor public class Course { @Id private String id; private String title; private String description; @CreatedDate private Instant createdAt; }
📥
CourseRequestDTO
Input DTO · Bean Validation
Unchanged from PostgreSQL version. @NotBlank rejects null/empty. @Size(max=150) enforces length. Validation happens at the controller before any service code runs.
@Data @NoArgsConstructor public class CourseRequestDTO { @NotBlank @Size(max=150) private String title; @NotBlank @Size(max=1000) private String description; }
📤
CourseResponseDTO
Output DTO · 201 Created
courseId changed from Long to String to match MongoDB ObjectId. Everything else unchanged. Decouples API contract from internal document model.
@Data @AllArgsConstructor public class CourseResponseDTO { private String courseId; // was Long private String title; private String description; private Instant createdAt; }

§ 07 — Deployment
Deploy to VPS
1
Build the Spring Boot JAR locally
Package the application skipping tests to generate the executable jar file.
./mvnw clean package -DskipTests # output: target/course-domain-0.0.1-SNAPSHOT.jar
2
Copy the JAR to the Droplet
Upload the built jar to the app folder on the VPS using SCP.
scp target/course-domain-0.0.1-SNAPSHOT.jar \ root@165.245.223.137:/opt/course-domain/course-domain.jar
3
Start the Spring Boot service on VPS
SSH into the Droplet and start the systemd service. It will auto-restart on reboot.
ssh root@165.245.223.137 systemctl start course-domain systemctl status course-domain
4
Build and deploy the frontend
Build Angular/React locally then copy the dist folder to the VPS frontend directory.
# Angular ng build --configuration production scp -r dist/* root@165.245.223.137:/opt/course-frontend/ # React npm run build scp -r build/* root@165.245.223.137:/opt/course-frontend/
5
Configure Nginx reverse proxy
Route traffic: / → frontend, /api/ → Spring Boot :8080, /node/ → Node.js :3000
server { listen 80; server_name 165.245.223.137; location / { root /opt/course-frontend; try_files $uri $uri/ /index.html; } location /api/ { proxy_pass http://localhost:8080; } location /node/ { proxy_pass http://localhost:3000; } }
